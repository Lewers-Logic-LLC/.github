## 1. Brand Identity Overview

Lewers Logic LLC is a **technology-forward, business-focused** brand that balances engineering precision with corporate trust. The brand serves enterprise and professional audiences who value innovation, reliability, and premium quality.

### Core Brand Pillars
- **Engineering Excellence**: Precision, logic, and technical sophistication
- **Premium Business**: Trust, professionalism, established authority
- **Accessible Innovation**: Technology made usable and clear

## 2. Logo and Visual Identity

### Logo Description
The Lewers Logic mark features:
- **"L" Letterform**: Primary teal accent (#25A09C) on the "L"
- **"Logic" Wordmark**: Deep Logic Blue (#081634)
- **Proportions**: Golden ratio alignment, geometric precision
- **Usage**: Always includes accompanying wordmark in primary applications

### Logo Variations
- **Primary (Full Color)**: Teal "L" + Deep Blue text (preferred)
- **Monochrome (Dark)**: All Deep Logic Blue (#081634)
- **Monochrome (Light)**: All Soft Gray (#E9F0EF)
- **Reverse**: On dark backgrounds, use all white or Soft Gray

### Clear Space
Minimum clear space around logo: 0.5x the height of the "L"

### Do's and Don'ts
✅ **Do:**
- Use primary full-color version on light backgrounds
- Scale proportionally (maintain aspect ratio)
- Apply monochrome versions for single-color contexts
- Use reverse/light on Dark backgrounds

❌ **Don't:**
- Stretch, compress, or distort the mark
- Change the teal "L" to gold (teal is reserved for the logo accent)
- Rotate or skew the letterform
- Add drop shadows, gradients, or effects not in the original
- Use on backgrounds with insufficient contrast

| **Near Black** | #12131A | 18, 19, 26 | Deepest applications, maximum contrast | Use sparingly; creates emphasis |
| **Navy Light** | #0D2B49 | 13, 43, 73 | Secondary dark, hover states on blue | Transition between blue and deeper states |

### Accent Colors (Equal Secondary)

| Color | Hex | RGB | Role | Usage |
|-------|-----|-----|------|-------|
| **Engineered Teal** | #25A09C | 37, 160, 156 | Primary Accent — Technology | Links, CTAs, form focus, interactive elements, product highlights, logo "L" |
| **Premium Gold Accent** | #C4962C | 196, 150, 44 | Secondary Accent — Business | Business context, trust signals, premium labels, pricing, awards, section dividers on formal docs |

### Accent Tonal Variations

**Teal Family:**
- **Teal Light** (#9DD2D0): Hover/highlight states, light backgrounds, legibility on dark
- **Teal Dark** (#318C88): Pressed/active states, deeper emphasis
- **Teal Range**: Provides depth and interaction feedback

**Gold Family:**
- **Gold Light** (#E0C97A): Hover/highlight states, legibility on dark backgrounds
- **Gold Dark** (#A67A1E): Pressed/active states, deeper emphasis
- **Gold Range**: Premium warmth without overwhelming

### Neutral Colors

| Color | Hex | RGB | Usage |
|-------|-----|-----|-------|
| **Soft Gray** | #E9F0EF | 233, 240, 240 | Light backgrounds, cards, subtle containers |
| **Gray Dark** | #727578 | 114, 117, 120 | Captions, metadata, disabled text, secondary content |

### Color Combinations (Do's and Don'ts)

✅ **Approved Combinations:**
- Teal on Deep Blue or white
- Gold on white or light gray backgrounds
- Teal text on white (links, CTAs)
- Gold text on white (premium labels, badges)
- Teal Light on Deep Blue (interactive hover on dark backgrounds)
- Gold Light on Deep Blue (premium hover on dark backgrounds)

❌ **Forbidden Combinations:**
- Teal text on gold background (insufficient contrast, visual clash)
- Gold text on teal background (insufficient contrast, visual clash)
- Teal + Gold in the same element (confusing, violates accent roles)
- Near Black text on Deep Blue (insufficient contrast)

### Color Accessibility
- **Contrast Ratios**: All text meets WCAG AA (4.5:1 minimum)
- **Dark Mode**: Use Teal Light (#9DD2D0) and Gold Light (#E0C97A) on dark backgrounds
- **Color Blindness**: Avoid relying solely on teal/gold distinction; always pair with icons, labels, or patterns

## 4. Typography

### Font Stack

**Primary Font: Inter**
- **Typographic Purpose**: Body, UI, navigation, general text
- **Source**: Google Fonts (Variable font preferred)
- **Weights Used**: 400 (Regular), 500 (Medium), 600 (Semibold), 700 (Bold)
- **Characteristics**: Modern, geometric, highly legible, technical-friendly

**Code/Monospace Font: JetBrains Mono**
- **Typographic Purpose**: Code blocks, technical content, terminal output
- **Source**: Google Fonts
- **Weights Used**: 400 (Regular), 600 (Semibold)
- **Characteristics**: Clear character distinction, technical credibility

### Type Scale

| Usage | Size | Weight | Line Height | Letter Spacing |
|-------|------|--------|-------------|-----------------|
| **H1 Heading** | 48px | 700 | 1.2 | -0.5px |
| **H2 Heading** | 36px | 700 | 1.3 | -0.3px |
| **H3 Heading** | 28px | 600 | 1.35 | 0px |
| **H4 Heading** | 22px | 600 | 1.4 | 0px |
| **Body Large** | 18px | 400 | 1.6 | 0px |
| **Body Regular** | 16px | 400 | 1.6 | 0px |
| **Body Small** | 14px | 400 | 1.5 | 0px |
| **Caption** | 12px | 400 | 1.4 | 0.5px |
| **Code Block** | 14px | 400 | 1.6 | 0px (mono) |

### Typography Usage

**Headings (H1–H4):**
- Color: Deep Logic Blue (#081634)
- Font-weight: 600–700
- Line-height: 1.2–1.4 (tighter for visual hierarchy)
- Letter-spacing: Negative or 0 (tighter tracking)
- Margin-bottom: 1rem (below heading)

**Body Text:**
- Color: Deep Logic Blue (#081634) or Navy Light (#0D2B49)
- Font-weight: 400
- Line-height: 1.6 (generous for readability)
- Font-size: 16px (primary), adjust as needed
- Max-width: 75 characters (optimal reading line)

**Captions & Metadata:**
- Color: Gray Dark (#727578)
- Font-weight: 400
- Font-size: 12–14px
- Text-transform: Occasionally uppercase for labels (with +0.5px tracking)

**Code/Monospace:**
- Font-family: JetBrains Mono, monospace
- Font-size: 14px
- Color: Near Black (#12131A) or Deep Logic Blue
- Background: Soft Gray (#E9F0EF)
- Padding: 2px 6px (inline), 16px (blocks)

**Links:**
- Color: Engineered Teal (#25A09C)
- Text-decoration: Underline (standard)
- Hover: Teal Dark (#318C88), maintain underline
- Active/Visited: Teal Dark (#318C88)

## 5. Accent Usage Rules (Critical)

### Teal (#25A09C) — Technology, Product, Interactive

**Use teal for:**
- Product UI and interactive elements
- Links and hypertext
- CTA buttons (primary action — "Submit", "Save", "Continue")
- Form focus rings and active states
- Icon accents in technology contexts
- Navigation highlights and active tabs
- Progress indicators and status signals
- Logo "L" (immutable)

**Rationale:** Teal conveys technology, innovation, and user action. It draws attention to interactive elements.

**Example contexts:** Software features, technical documentation, form interactions, product demos.

### Gold (#C4962C) — Business, Trust, Premium

**Use gold for:**
- Business context and corporate branding
- Trust signals and certifications (badges, seals)
- Premium labels ("Premium", "Exclusive", "Enterprise")
- Pricing highlights and premium tiers
- Section dividers on formal/corporate documents
- Awards and achievement badges
- "Learn More" or secondary call-to-action buttons
- Premium pricing labels and tier indicators
- Official certificates and credentials

**Rationale:** Gold conveys value, prestige, and business authority. It signals premium quality and trust.

**Example contexts:** Enterprise pricing pages, corporate documents, award announcements, business certifications.

### Button Usage (Both Accents)

**Primary Buttons (Teal):**
- Action verbs: Submit, Save, Continue, Send, Create, Update
- Technology-focused actions
- Primary call-to-action on product pages
- Form submission buttons
- Default state: Engineered Teal (#25A09C)
- Hover: Teal Dark (#318C88)
- Active/Pressed: Teal Dark (#318C88) + slight inset effect

**Secondary Buttons (Gold):**
- "Learn More", "Explore", "Discover" (informational CTAs)
- Premium or upsell actions
- Business/corporate contexts
- Alternative actions on pricing pages
- Default state: Premium Gold Accent (#C4962C)
- Hover: Gold Light (#E0C97A)
- Active/Pressed: Gold Dark (#A67A1E) + slight inset effect

### Interactive States

| State | Teal | Gold |
|-------|------|------|
| **Default** | #25A09C | #C4962C |
| **Hover** | #318C88 | #E0C97A |
| **Active/Pressed** | #318C88 (darker bg) | #A67A1E |
| **Disabled** | Gray Dark (#727578) | Gray Dark (#727578) |
| **Focus Ring** | Teal Light (#9DD2D0) on dark bg | Gold Light (#E0C97A) on dark bg |

### Dark Background Usage

When placing accents on Deep Logic Blue or dark backgrounds:
- **Teal**: Use Teal Light (#9DD2D0) for legibility
- **Gold**: Use Gold Light (#E0C97A) for legibility
- **Contrast**: Both meet WCAG AA standards on Dark backgrounds

## 6. Spacing and Layout

### Spacing Scale (8px System)

All spacing increments are multiples of 8px for consistent rhythm and alignment.

| Size | Pixels | Usage |
|------|--------|-------|
| xs | 4px | Minimal padding, icon spacing |
| sm | 8px | Padding within compact elements |
| md | 16px | Standard padding, default spacing |
| lg | 24px | Heading spacing, section gaps |
| xl | 32px | Major section breaks |
| 2xl | 48px | Large whitespace, page sections |
| 3xl | 64px | Full-page spacing |

### Application
- **Container padding**: 16px–24px
- **Component padding**: 8px–16px
- **Margin between sections**: 24px–48px
- **Margin between elements**: 8px–16px
- **Grid gap**: 16px (default), 24px (larger layouts)

## 7. Border Radius

### Radius Scale

| Size | Pixels | Usage |
|------|--------|-------|
| **Minimal** | 2px | Slight rounding, technical elements |
| **Small** | 4px | Buttons, small components |
| **Medium** | 8px | Cards, medium containers |
| **Large** | 12px | Modals, large sections |
| **Full** | 50% (circle) | Avatars, badges, circular elements |

### Application Rules
- **Buttons**: 4–8px (medium button size: 8px)
- **Cards**: 8px (standard), 12px (featured)
- **Modals & Overlays**: 12px
- **Icons**: 4–8px (avoid over-rounding)
- **Form inputs**: 4–6px
- **Images**: 0–8px (context dependent)

## 8. Shadows and Elevation

### Shadow System

| Level | Box-shadow | Usage |
|-------|-----------|-------|
| **Elevation 1** | 0 2px 4px rgba(8, 22, 52, 0.1) | Subtle lift, hover states |
| **Elevation 2** | 0 4px 8px rgba(8, 22, 52, 0.15) | Cards, button hover |
| **Elevation 3** | 0 8px 16px rgba(8, 22, 52, 0.2) | Modals, overlays |
| **Elevation 4** | 0 12px 24px rgba(8, 22, 52, 0.25) | Maximum emphasis (rare) |
| **Inset** | inset 0 1px 2px rgba(8, 22, 52, 0.1) | Pressed buttons, inputs |

### Application
- **Cards**: Elevation 2 (default), Elevation 3 (on hover)
- **Buttons**: Elevation 1 (subtle), Elevation 2 (hover)
- **Modals**: Elevation 4
- **Pressed State**: Inset shadow + slight downward shift
- **Focus Ring**: No shadow, use color outline instead

## 9. Voice and Tone

### Brand Voice Characteristics
- **Precise**: Clear, logical, unambiguous language
- **Professional**: Authoritative, business-appropriate
- **Approachable**: Not cold; warm professionalism
- **Confident**: Assured without arrogance
- **Technical Yet Accessible**: Expert knowledge made understandable

### Tone Variations

**Formal (Corporate, Legal, Enterprise):**
- Use: Business announcements, contracts, official communications
- Tone: Authoritative, precise, professional
- Example: "Lewers Logic LLC provides enterprise-grade solutions for mission-critical systems."

**Approachable (Product, User-Facing):**
- Use: Product copy, onboarding, help text
- Tone: Clear, friendly, professional
- Example: "Let's set up your account so you can start building smarter systems."

**Technical (Documentation, Dev-Focused):**
- Use: API docs, technical guides, code examples
- Tone: Precise, detailed, logical
- Example: "Initialize the SDK using the provided credentials and call the authenticate() method."

### Writing Guidelines

✅ **Do:**
- Use active voice
- Keep sentences short and clear
- Be specific and concrete
- Use technical terms accurately
- Address the reader directly ("you")
- Write for scannability (headings, bullets)
- Use consistent terminology

❌ **Don't:**
- Use corporate jargon unnecessarily
- Write vague generalities
- Use passive voice excessively
- Overcomplicate simple concepts
- Use ALL CAPS (except for acronyms)
- Write in first-person plural without purpose ("we offer…" is fine; "we believe…" is vague)

## 10. Design System Do's and Don'ts

### Do's

✅ **Color**
- Do use Deep Logic Blue as the dominant color for structure
- Do use teal for interactive elements and product features
- Do use gold for business context and premium signals
- Do maintain sufficient contrast (WCAG AA minimum)
- Do test color combinations for colorblind accessibility
- Do use the accent tonal variations (Light/Dark) for interactive states

✅ **Typography**
- Do use Inter for all body and UI text
- Do use JetBrains Mono for code
- Do maintain consistent type scales across layouts
- Do use bold weights (600+) for headings only
- Do pair headings and body text with adequate line-height
- Do use captions sparingly and consistently

✅ **Layout**
- Do use the 8px spacing scale consistently
- Do use border-radius values from the scale
- Do apply shadows for depth and hierarchy
- Do test on responsive breakpoints (mobile, tablet, desktop)
- Do align elements to grids or spacing scale

✅ **Branding**
- Do use the primary logo variation in most contexts
- Do apply the brand color palette correctly
- Do maintain clear space around the logo
- Do test brand applications in context before deployment
- Do document any extensions or custom uses

### Don'ts

❌ **Color**
- Don't use teal + gold in the same element
- Don't place teal text on gold backgrounds
- Don't place gold text on teal backgrounds
- Don't introduce unauthorized colors
- Don't use color as the only distinguishing factor for information
- Don't use full-saturation colors outside the approved palette

❌ **Typography**
- Don't use fonts other than Inter or JetBrains Mono
- Don't use multiple typefaces in a single layout
- Don't set body text below 14px or above 18px without reason
- Don't apply multiple font-weights excessively
- Don't use text-transform: uppercase for body text
- Don't use decorative fonts or styles (all caps body text, excessive letter-spacing)

❌ **Layout**
- Don't ignore the 8px spacing scale
- Don't apply border-radius values outside the scale
- Don't create elements with insufficient spacing
- Don't stretch or compress the logo
- Don't place content too close to page edges
- Don't use conflicting shadow systems

❌ **Branding**
- Don't modify the logo mark
- Don't change the teal "L" to any other color
- Don't use brand colors without contrast consideration
- Don't apply branding inconsistently across touchpoints
- Don't use the monochrome logo on the primary website
- Don't scale logos below 48px at small sizes

## 11. Implementation Checklist

Before launching a new brand application, verify:

- [ ] Logo is primary variation on light backgrounds, reverse/monochrome on dark
- [ ] All body text uses Inter or JetBrains Mono
- [ ] Color palette matches hex values exactly
- [ ] All text meets WCAG AA contrast (4.5:1 minimum)
- [ ] Spacing follows 8px system
- [ ] Border radius uses approved scale
- [ ] Teal accents used for interactive elements only
- [ ] Gold accents used for business/premium context only
- [ ] No teal+gold combinations in same elements
- [ ] Teal/Gold never combined with incompatible backgrounds
- [ ] Dark backgrounds use Teal Light (#9DD2D0) and Gold Light (#E0C97A)
- [ ] Voice and tone are consistent with brand values
- [ ] All interactive states (hover, active, focus) are defined
- [ ] Responsive design tested across breakpoints
